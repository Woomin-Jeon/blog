## Findings

- **git 충돌시 해결 흐름**  
  
    ```bash
    $ git fetch upstream master # 원격 레포지토리의 데이터 가져오기
    $ git rebase upstream/master # 내 브랜치에 병합

    # 충돌!

    # 파일들을 살펴보며 충돌한 내용들 수정

    $ git add . # 수정한 파일들 staged
    $ git rebase --continue # 커밋

    # 만약 git이 꼬여서 rebase 전으로 되돌리고자 한다면
    $ git rebase --abort
    ```
